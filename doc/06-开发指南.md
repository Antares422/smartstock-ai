## 3. 代码规范

### 3.1 Java 代码规范

#### 3.1.1 命名规范

- **类名**：大驼峰命名法（PascalCase），如 `UserService`、`StockController`
- **方法名**：小驼峰命名法（camelCase），如 `getUserById`、`calculateProfit`
- **变量名**：小驼峰命名法，如 `userId`、`stockCode`
- **常量名**：全大写 + 下划线，如 `MAX_RETRY_COUNT`、`DEFAULT_PAGE_SIZE`
- **包名**：全小写，如 `com.smartstock.service`

#### 3.1.2 注释规范

**类注释**：
```java
/**
 * 用户服务类
 *
 * @author 开发者姓名
 * @since 2026-03-09
 */
public class UserService {
    // ...
}
```

**方法注释**：
```java
/**
 * 根据用户 ID 获取用户信息
 *
 * @param userId 用户 ID
 * @return 用户信息
 * @throws UserNotFoundException 用户不存在时抛出
 */
public User getUserById(Long userId) {
    // ...
}
```

**行内注释**：
```java
// 计算手续费（买入金额 × 0.03%，最低 5 元）
BigDecimal fee = amount.multiply(new BigDecimal("0.0003")).max(new BigDecimal("5"));
```

#### 3.1.3 代码格式

- **缩进**：4 个空格（不使用 Tab）
- **行宽**：不超过 120 字符
- **空行**：方法之间空一行，逻辑块之间空一行
- **大括号**：左大括号不换行，右大括号换行

**示例**：
```java
public class UserService {

    public User getUserById(Long userId) {
        if (userId == null) {
            throw new IllegalArgumentException("用户 ID 不能为空");
        }

        User user = userMapper.selectById(userId);
        if (user == null) {
            throw new UserNotFoundException("用户不存在");
        }

        return user;
    }
}
```

#### 3.1.4 异常处理

- 使用自定义异常类（继承 `RuntimeException`）
- 不要捕获 `Exception` 或 `Throwable`
- 不要使用空的 `catch` 块

**示例**：
```java
public class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(String message) {
        super(message);
    }
}

// 使用
if (user == null) {
    throw new UserNotFoundException("用户不存在：" + userId);
}
```

### 3.2 数据库规范

#### 3.2.1 SQL 编写规范

- 使用 MyBatis Plus 的 Lambda 查询，避免硬编码字段名
- 复杂查询使用 XML 方式编写 SQL
- 避免使用 `SELECT *`，明确指定字段
- 使用参数化查询，防止 SQL 注入

**示例**：
```java
// 推荐：使用 Lambda 查询
List<User> users = userMapper.selectList(
    new LambdaQueryWrapper<User>()
        .eq(User::getStatus, 1)
        .orderByDesc(User::getCreatedAt)
);

// 不推荐：硬编码字段名
List<User> users = userMapper.selectList(
    new QueryWrapper<User>()
        .eq("status", 1)
        .orderByDesc("created_at")
);
```

#### 3.2.2 索引使用规范

- 为常用查询字段添加索引
- 联合索引遵循最左前缀原则
- 避免在索引字段上使用函数

### 3.3 Git 规范

#### 3.3.1 分支管理

- **main**：主分支，保持稳定，只接受 merge
- **develop**：开发分支，日常开发在此分支
- **feature/xxx**：功能分支，开发新功能
- **bugfix/xxx**：修复分支，修复 bug
- **hotfix/xxx**：紧急修复分支，修复生产环境 bug

#### 3.3.2 提交规范

**提交消息格式**：
```
<type>(<scope>): <subject>

<body>

<footer>
```

**type 类型**：
- `feat`：新功能
- `fix`：修复 bug
- `docs`：文档更新
- `style`：代码格式调整（不影响功能）
- `refactor`：重构代码
- `test`：添加测试
- `chore`：构建工具或辅助工具的变动

**示例**：
```
feat(trade): 添加买入股票功能

- 实现买入订单创建
- 实现资金扣除逻辑
- 实现持仓更新逻辑

Closes #123
```

### 4.2 分层说明

- **Controller 层**：接收 HTTP 请求，参数校验，调用 Service 层
- **Service 层**：业务逻辑处理，事务管理
- **Mapper 层**：数据访问，与数据库交互

- **DTO 层**：数据传输对象，用于接口请求参数
- **VO 层**：视图对象，用于接口响应数据

## 5. 开发流程

### 5.1 功能开发流程

1. **需求分析**：阅读 PRD，理解需求
2. **技术设计**：设计数据库表、接口、类结构
3. **创建分支**：从 `develop` 分支创建 `feature/xxx` 分支
4. **编写代码**：按照代码规范编写代码
5. **单元测试**：编写单元测试，覆盖率 > 80%
6. **自测**：本地测试功能是否正常
7. **提交代码**：提交到 `feature/xxx` 分支
8. **代码审查**：提交 Pull Request，等待审查
9. **合并代码**：审查通过后，合并到 `develop` 分支
10. **集成测试**：在测试环境验证功能

### 5.2 Bug 修复流程

1. **Bug 确认**：确认 Bug 是否存在，复现步骤
2. **创建分支**：从 `develop` 分支创建 `bugfix/xxx` 分支
3. **定位问题**：使用日志、调试工具定位问题
4. **修复代码**：修复 Bug，添加测试用例
5. **自测**：验证 Bug 是否修复
6. **提交代码**：提交到 `bugfix/xxx` 分支
7. **代码审查**：提交 Pull Request，等待审查
8. **合并代码**：审查通过后，合并到 `develop` 分支

## 6. 测试规范

### 6.1 单元测试

#### 6.1.1 测试框架

- **JUnit 5**：单元测试框架
- **Mockito**：Mock 框架
- **AssertJ**：断言库

#### 6.1.2 测试规范

- 每个 Service 方法都应有对应的单元测试
- 测试方法命名：`test方法名_场景_期望结果`
- 使用 `@Mock` 模拟依赖
- 使用 `@InjectMocks` 注入被测试对象

**示例**：
```java
@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock
    private UserMapper userMapper;

    @InjectMocks
    private UserService userService;

    @Test
    void testGetUserById_UserExists_ReturnUser() {
        // Given
        Long userId = 1L;
        User mockUser = new User();
        mockUser.setId(userId);
        mockUser.setUsername("test@example.com");

        when(userMapper.selectById(userId)).thenReturn(mockUser);

        // When
        User result = userService.getUserById(userId);

        // Then
        assertThat(result).isNotNull();
        assertThat(result.getId()).isEqualTo(userId);
        assertThat(result.getUsername()).isEqualTo("test@example.com");
    }

    @Test
    void testGetUserById_UserNotExists_ThrowException() {
        // Given
        Long userId = 999L;
        when(userMapper.selectById(userId)).thenReturn(null);

        // When & Then
        assertThatThrownBy(() -> userService.getUserById(userId))
            .isInstanceOf(UserNotFoundException.class)
            .hasMessage("用户不存在");
    }
}
```

### 6.2 集成测试

#### 6.2.1 测试框架

- **Spring Boot Test**：集成测试框架
- **TestContainers**：容器化测试（MySQL、Redis）

#### 6.2.2 测试规范

- 使用 `@SpringBootTest` 启动完整的 Spring 上下文
- 使用 `@Transactional` 回滚测试数据
- 使用 `@AutoConfigureMockMvc` 测试 Controller

**示例**：
```java
@SpringBootTest
@AutoConfigureMockMvc
@Transactional
class UserControllerIntegrationTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    void testRegister_ValidInput_ReturnSuccess() throws Exception {
        String requestBody = """
            {
                "email": "test@example.com",
                "password": "password123",
                "nickname": "测试用户"
            }
            """;

        mockMvc.perform(post("/api/auth/register")
                .contentType(MediaType.APPLICATION_JSON)
                .content(requestBody))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.code").value(200))
            .andExpect(jsonPath("$.data.email").value("test@example.com"));
    }
}
```

## 7. 常见问题

### 7.1 数据库连接失败

**问题**：启动时报错 `Communications link failure`

**解决方案**：
1. 检查 MySQL 是否启动：`systemctl status mysql`
2. 检查数据库配置是否正确（用户名、密码、端口）
3. 检查防火墙是否阻止连接

### 7.2 Redis 连接失败

**问题**：启动时报错 `Unable to connect to Redis`

**解决方案**：
1. 检查 Redis 是否启动：`systemctl status redis-server`
2. 检查 Redis 密码是否正确
3. 使用 `redis-cli -a Mv27Yq0yv2 ping` 测试连接

### 7.3 Kafka 连接失败

**问题**：启动时报错 `Failed to connect to Kafka`

**解决方案**：
1. 检查 Kafka 是否启动：`systemctl status kafka`
2. 检查 Kafka 地址是否正确（`localhost:9092`）
3. 检查主题是否已创建
